# 미래에셋생명 전사 게시판 AI 활용 방안 제안서

## 배경

미래에셋생명은 **AI Native 보험사**를 전사 전략으로 추진하고 있습니다.
이를 위해 전체 임직원이 스스로 **"어떻게 나의 업무에 AI를 적용할 수 있을까?"**를 고민하고 실행하는 문화가 필요합니다.

현재 전사 게시판(Mi-SQUARE)은 이미 전 직원이 매일 접속하는 플랫폼입니다.
이 게시판을 활용하여 AI 확산의 3가지 핵심 과제를 해결하는 방안을 제안합니다.

---


## 타겟 1. "AI Native 보험사, 왜 해야 하지?" → AI 전략 소통

### 문제
- 경영진의 AI Native 전략이 현업 직원에게 충분히 전달되지 않음
- "왜 AI를 해야 하는지" 공감하지 못하면 자발적 참여는 일어나지 않음

### 게시판 해결 방안: 「AI 전략 & 비전」 게시판

| 콘텐츠 | 설명 | 게시 주기 |
|---|---|---|
| **CEO/CTO 메시지** | AI Native 보험사 비전과 방향성을 경영진이 직접 전달 | 월 1회 |
| **AI 전략 해설** | AI 혁신팀이 전략을 쉽게 풀어서 설명 (Q&A 형식) | 격주 |
| **업계 동향** | 타 보험사/금융사의 AI 도입 사례, 시장 변화 | 주 1회 |
| **AX 추진 현황** | 전사 AI 프로젝트 진행 상황, 성과 공유 | 월 1회 |

### 기대 효과
- 전략의 **당위성** 공감 → "남의 일"에서 "나의 일"로 인식 전환
- 업계 동향을 통한 **위기감 + 기회 인식** 동시 자극

---

## 타겟 2. "AI가 뭔데?" → AI Champion 교육

### 문제
- AI 자체에 대한 이해가 부족한 직원이 다수
- 일회성 교육으로는 실질적 역량 향상이 어려움

### 게시판 해결 방안: 「AI Champion 교육」 게시판

#### AI Champion 프로그램이란?

각 부서에서 선발된 **AI 활용 선도자**를 체계적으로 육성하고,
이들이 부서 내 AI 확산을 주도하는 **전파자 모델**입니다.

```
┌─────────────────────────────────────────────────┐
│              AI Champion 교육 여정               │
│                                                 │
│  Step 1        Step 2        Step 3       Step 4│
│  ┌─────┐      ┌─────┐      ┌─────┐     ┌─────┐│
│  │기초  │ ──→ │실전  │ ──→ │공유  │ ──→ │인증  ││
│  │교육  │      │과제  │      │활동  │     │부여  ││
│  └─────┘      └─────┘      └─────┘     └─────┘│
│  AI 개념       업무 적용     꿀팁/Q&A    Champion│
│  도구 사용법    성과 도출     사례 작성    배지 부여│
└─────────────────────────────────────────────────┘
```

#### 게시판 구성

| 서브 게시판 | 역할 | 상세 |
|---|---|---|
| **교육 공지** | 교육 일정/신청 안내 | Champion 기수별 모집 공고, 교육 일정, 수료 기준 안내 |
| **학습 자료** | AI 기초~실전 콘텐츠 | 난이도별 교육 자료 (입문/중급/고급), VOD 강의, 외부 교육 링크 |
| **AI 도구 가이드** | 사내 승인 도구 안내 | 승인된 AI 도구 목록, 사용 매뉴얼, 보안 유의사항 |
| **Champion 활동** | 수료자 활동 공간 | Champion 소개, 기수별 활동 현황, 우수 Champion 시상 |

#### AI Champion의 역할 (교육 이수 후)

| 역할 | 활동 | 효과 |
|---|---|---|
| **부서 내 전파자** | 부서원 대상 미니 교육, 도구 사용법 안내 | AI를 모르는 직원의 진입 장벽 해소 |
| **성공사례 생산자** | 본인의 AI 적용 결과를 게시판에 공유 | 실전 사례 축적 |
| **Q&A 답변자** | 동료 질문에 답변 (Champion 배지 표시) | 자체 해결 역량 확보 |
| **혁신팀 연결고리** | 현장 니즈/피드백을 AI 혁신팀에 전달 | 현업-전략 간 간극 해소 |

### 기대 효과
- "AI를 모른다" → Champion이 **1:1 전파자**가 되어 부서별 맞춤 교육
- 일방적 교육이 아닌 **동료 학습(Peer Learning)** 기반의 자연스러운 확산

## 타겟 3. "내 업무에 어떻게 쓰지?" → 실전 적용 지원

### 문제
- AI를 알아도 **본인 업무에 어떻게 적용할지** 막막함
- 기술적 지원이 필요한 경우와 외부 도구로 바로 해결 가능한 경우가 혼재

### 해결 방안은 두 가지 트랙으로 나뉩니다:

---

### 타겟 3-1. 기술적 도움이 필요한 경우 → 「AI 혁신팀 프로젝트」 게시판

> AI 혁신팀이 현업 부서와 함께 AI 프로젝트를 수행하는 공식 채널

| 서브 게시판 | 역할 | 상세 |
|---|---|---|
| **프로젝트 소개** | 진행 중인 AI 프로젝트 공유 | 프로젝트 배경, 목표, 참여 부서, 진행 상황 |
| **프로젝트 의뢰** | 현업 부서의 AI 프로젝트 요청 | 템플릿 기반 의뢰서 작성 (업무 설명, 기대 효과, 희망 일정) |
| **성공사례** | 완료된 프로젝트 결과 공유 | Before/After 비교, 효과 수치, 적용 과정, 교훈 |

#### 프로젝트 의뢰 → 수행 흐름

```
현업 부서                    AI 혁신팀
   │                           │
   │  ① 프로젝트 의뢰서 작성     │
   │  (게시판 템플릿)            │
   │ ──────────────────────→  │
   │                           │  ② 검토 & 우선순위 판단
   │                           │
   │  ③ 킥오프 미팅              │
   │ ←──────────────────────  │
   │                           │
   │  ④ 공동 수행                │
   │ ←─────────────────────→  │
   │                           │
   │  ⑤ 성공사례 게시판에 결과 공유 │
   │ ──────────────────────→  │
```

---

### 타겟 3-2. 외부 AI로 바로 해결 가능한 경우 → 「AI 활용 커뮤니티」 게시판

> 직원들이 직접 외부 AI 도구를 활용한 경험을 공유하고 도움을 주고받는 공간

| 서브 게시판 | 역할 | 상세 |
|---|---|---|
| **AI 꿀팁 공유** | 실전 노하우 공유 | 프롬프트 팁, 업무 자동화 방법, 유용한 AI 도구 소개 |
| **AI Q&A** | 실시간 질의응답 | AI 도구 사용 중 궁금한 점, Champion이 우선 답변 |
| **AX 아이디어 제안** | 상향식 혁신 제안 | AI 적용 아이디어 제안, 투표로 우선순위 결정 |
| **도구 활용 지원** | 외부 AI 도구 지원 | 도구 도입 요청, 라이선스 문의, 사용 승인 절차 안내 |

### 왜 좌측 사이드바 게시판 형태인가?

| 기준 | 판단 |
|---|---|
| **개발 비용** | 기존 게시판 구조 그대로 활용 → 최소 개발 |
| **학습 비용** | 직원들이 이미 익숙한 게시판 UI → 바로 참여 가능 |
| **속도** | 즉시 오픈 가능 (상단 탭 추가보다 빠름) |
| **접근성** | 매일 사용하는 게시판 영역에 위치 → 자연스러운 노출 |
| **확장성** | 서브 게시판 추가/삭제가 자유로움 |

## 3가지 타겟의 연계 구조

각 게시판은 독립적으로 운영되지만, **AI Champion**이 연결고리가 되어 선순환을 만듭니다.

```
        ┌──────────────────────────────────────────┐
        │         타겟 1. AI 전략 소통               │
        │  "왜 AI를 해야 하는가" → 동기 부여           │
        └──────────────┬───────────────────────────┘
                       │ 동기 부여된 직원
                       ▼
        ┌──────────────────────────────────────────┐
        │         타겟 2. AI Champion 교육           │
        │  "AI란 무엇인가" → 역량 확보                │
        └──────────────┬───────────────────────────┘
                       │ 역량을 갖춘 Champion
                       ▼
        ┌──────────────────────────────────────────┐
        │         타겟 3. 실전 적용 지원              │
        │                                          │
        │  3-1. 복잡한 과제 → AI 혁신팀 프로젝트       │
        │  3-2. 간단한 활용 → 외부 AI + 커뮤니티       │
        └──────────────┬───────────────────────────┘
                       │ 성공사례 생성
                       ▼
              ┌────────────────┐
              │  성공사례 공유   │
              │  → 타겟1 강화   │ ← 선순환
              │  → 다른 직원    │
              │    동기 부여    │
              └────────────────┘
```

## 운영 주체별 역할 분담

| 게시판 | 운영 주체 | 콘텐츠 생산자 | 대상 |
|---|---|---|---|
| **AI 전략** | AI 혁신팀 + 경영진 | 경영진, AI 혁신팀 | 전 직원 |
| **AI 교육** | AI 혁신팀 | AI 혁신팀, 외부 강사 | 교육 대상자, Champion 후보 |
| **AI 프로젝트** | AI 혁신팀 | AI 혁신팀 + 현업 부서 | 프로젝트 의뢰 부서 |
| **AI 커뮤니티** | 전 직원 (자율) | AI Champion + 전 직원 | 전 직원 |

## 도입 로드맵

| 단계 | 시기 | 내용 | 비고 |
|---|---|---|---|
| **Phase 1** | 1~2주차 | AI 커뮤니티 게시판 개설 (꿀팁, Q&A, 아이디어) | 기존 게시판 구조 활용, 즉시 오픈 |
| **Phase 2** | 3~4주차 | AI 전략 게시판 개설 + 첫 CEO 메시지 게시 | 전략 공감 형성 시작 |
| **Phase 3** | 5~6주차 | AI 교육 게시판 개설 + Champion 1기 모집 | 교육 프로그램 시작 |
| **Phase 4** | 7~8주차 | AI 프로젝트 게시판 개설 + 첫 프로젝트 의뢰 접수 | 실전 적용 채널 가동 |
| **Phase 5** | 9주차~ | 전체 운영 안정화, Champion 2기 모집, 성과 분석 | 선순환 구조 확립 |

> **Phase 1을 커뮤니티부터 시작하는 이유:**
> 가장 가벼운 참여(꿀팁 공유, Q&A)부터 시작하여 직원들의 거부감을 낮추고,
> 게시판 활용 습관을 먼저 형성한 뒤 교육 및 프로젝트 채널을 순차적으로 오픈합니다.

## 기대 효과 요약

| 타겟 | 해결 방안 | 기대 효과 |
|---|---|---|
| **1. 전략을 모른다** | AI 전략 게시판 | 경영진 메시지 + 업계 동향으로 "왜 해야 하는지" 공감 |
| **2. AI를 모른다** | AI Champion 교육 게시판 | 체계적 교육 + 부서별 전파자 육성으로 전사 역량 향상 |
| **3-1. 적용이 어렵다 (복잡)** | AI 프로젝트 게시판 | AI 혁신팀과 현업이 함께 프로젝트 수행 → 실질적 성과 |
| **3-2. 적용이 어렵다 (간단)** | AI 커뮤니티 게시판 | 동료 간 노하우 공유 + 도구 지원으로 즉시 활용 가능 |
